# Emoji Picker (Qt / KDE Wayland)

A fast **native emoji picker for KDE Wayland**, inspired by the Windows `Win + .` emoji panel.

The picker opens quickly, lets you search emojis by shortcode or tags, queue multiple emojis, and paste them into the previously focused application.

The goal of this project is **simplicity, speed, and a keyboard-first workflow**.

## Features

- Fast startup
- Search emojis by **shortcode or tags**
- Queue multiple emojis before inserting
- Paste queued emojis into the previously focused application
- Fully keyboard-driven
- Proper Unicode grapheme deletion (removes full emoji sequences)
- Wayland-compatible paste using `wl-copy` + `ydotool`
- Lightweight Qt/QML UI

---

## Keybinds

| Key | Action |
|---|---|
| `Enter` | Add selected emoji to queue |
| `Ctrl + W` or `Shift + Backspace` | Remove last emoji from queue |
| `Ctrl + U` | Clear queue |
| `Esc` | Paste queue and close |
| `Ctrl + Q` or `Shift + Esc` | Close without pasting |

## Requirements

Runtime dependencies:

- Qt6
- `wl-clipboard` (`wl-copy`)
- `ydotool`

Example install on Arch Linux:

```bash
sudo pacman -S qt6-base qt6-declarative wl-clipboard ydotool
```

## Emoji Cache

The picker reads emojis from a cache file:

```
~/.cache/emoji-picker/emoji.list
```

This file is generated by a companion repository.

### Cache Builder

Generate the cache using:

https://github.com/qtov/emoji-picker-cache

The builder downloads the **GitHub emoji dataset (`gemoji`)** and creates a compact searchable list.

Example cache entry:

```
🤣 :rofl: rofl rolling_on_the_floor_laughing lol lmao
```

## KDE Shortcut Setup

Create a global shortcut that runs:

```bash
emoji-picker --toggle
```

Example:

```
Meta + .
```

Due to needing `--toggle` you also need to create the app as such.

Location: `~/.local/share/applications/emoji-picker.desktop`
```
[Desktop Entry]
Type=Application
Name=Emoji Picker
Exec=emoji-picker --toggle
Icon=face-smile
Terminal=false
Categories=Utility;
```

After which, update kde app list:

`update-desktop-database ~/.local/share/applications`

## How It Works

1. Picker loads the cached emoji list
2. User searches or navigates the emoji list
3. Selected emojis are added to a queue
4. When inserting:
   - emojis are copied with `wl-copy`
   - paste is simulated via `ydotool`
   - the picker window closes

This avoids relying on compositor-specific protocols and works reliably on Wayland.
